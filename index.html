HTML

  • The head element contains inforation about the webpage
  • The body element represents the visible content shown to the user
  • h(n): heading; larger, more prominent titles and subtitles to organize a page. They can span from H1-H6
  • p: paragraph; these elements represent a block of text
  • ul: unordered list, marks its elements with bullet prominent
  • ol: ordered list, marks its elements with numbers
  • br: break; creates a line of empty space, or a line break between two block of content
  • a: anchor; creaks links to the same or other webpages; contains an href attribute which points to the URL for the link

CSS

  • CSS imparts style to your webpage using declarations
  • any line of CSS code consist of a selector and a declaration
  • a selector defintes the element or attributes to which the rules or declarations will apply
    • "div" is a common selector placeholder
    • however, you can use any HTML element as a selector to apply rules to them
  • declarations consist of 2 important components
    • The CSS property we want to apply (e.g. color)
    • the value of said property
    • e.g. div {color: blue; text align: justify;}
  • The box method describes the relation of objects in your webpage
  • There are multiple different selectors we can choose to exercise rules to different elements
  • Examples of HTML elements you can make rules for:
      • Examples of HTML elements you can make rules for:
    • *: applies a given rule to all the elements on your HTML page
    • header: applies rules to the header
    • footer: applies rules to the footer
    • hx: where x = 1-6 to apply rules to headers
    • p: applies a set of rules across all paragraphs in the file
    • In order to save time and not repeat yoself, you can apply the same rules to different selectors by simply separating them with a comma
    • Additionally, you can assign classes to HTML elements by adding "class=x" in their tag

    GIT

    • git init: initiates an empty repo, use this when you first enter a directory to create a .git file
    • git status: see what is set for staging and what working files need to be updated/added/removed
    • git add <filename>: add a file to staging
    • git add -u: updates all the files needed in staging (quick hand so as to not add files separately)
    • git commit -m <commit message here>: make the commit message as descriptive as possible; takes a snapshot of the repo if you were to push the branch this is what you would be "pushing"
    • git push: pushes what file you're on to the repo
    • git branch -b <newBranchName>: creates a new branch
    • git checkout<branchName>: takes you to a certain branch
    • git pull: pull down changes from repo, comes in handy when you work with others
    • git branch -a: display all branches that are in the repo before pull

    Javascript

    • Javascript makes a website interactive
    • A variable is a named container that allows us to store values in our code.
      • e.g. let band = Creeping Death"
      • For the above example, console.log(band) will display "Creeping Death" in your browser console
    • Control flow is the order in which a computer executes code in a script, it always runs linearly, from top to bottom
      • Because of control flow, the link to JS within an html file should always be inserted at the bottom of the body
    • conditional statements and loops control the order in which code is executed
    • if statements are a type of conditional statement that will execute a code if the statment meets a specific condition
      • if(condition) {}
      • you can set up more than one condition by using else if (second condition) {}
    • An array is a single variable that is used to hold a group of data
      • let bands = ["Creeping Death", "200 Stab Wounds", "Undeath"];
      • arrays are indexed at zero, meaning in the above, Creeping Death would have a value of 0
    • for loops harness the index order, making it possible to perform a task on all the items in an array
      • for loops contain 3 important statements
        1. the starting point of our loop let x = 0
        2. the condition which allows the loop to execute code once per item in the array; x < bands.length;
        3. the final statement which allows the array to iterate over each individual item; x++
      • e.g.: let(bands x = 0; x < shapes.length; x++) { // code block }
    • a function: is a set of instructions that informs the computer how to perform a certain task
      • Unlike for loops, they are not automatically executed, and instead have to be called by name
      • to create a function, use first use the function keyword to declare a new variable and give our functin a unique name followed by ()
      • follow this line with {}, which will contain the code you want to execute
      • in order to call a function, simply type out the appropriate function name with ()
